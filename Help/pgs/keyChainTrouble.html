Troubleshooting

If you don't find what you're looking for here, also take a look at the FAQs page. Some possible problems and solutions are:

"Keychain item could not be created" or "Keychain item could not be edited" Error message
KeyChainDD uses the OS X keychain management infrastructure. This error message usually indicates that there is a problem with that. Try (a) fixing file permissions, and (b) using the OS X KeyChain Access application's "Keychain First Aid" tool.
"OS X does not have a default Keychain location specified" Error message
This indicates that OS X reported not having a location to store keychains. This indicates a serious misconfiguration of OS X. Try (a) fixing file permissions, and (b) using the OS X KeyChain Access application's "Keychain First Aid" tool.
Getting help
You can post questions on the forums at KeyChainDD at SourceForge.net.
When all else fails.....
If KeyChainDD gets completely confused, it will output diagnostic/error messages to the Console. These messages may help you to find the root cause of the problem. Start the Console application, and repeat whatever steps led to the problem.
Reporting bugs
You can report bugs/problems via KeyChainDD's home at KeyChainDD at SourceForge.net. You should BOTH send me an e-mail via the SourceForge e-mail system, AND leave a message on the SourceForge KeyChainDD forum. Please send the crash log (if applicable) and any messages output to the Console.
