Using KeyChainDD securely

KeyChainDD manages all of your on-line passwords securely, allowing you to easily use strong, highly secure passwords, but still keep all the ease of use - like drag and drop - that the Mac OS X platform is known for. In addition, KeyChainDD supports double levels passwords - systems where you are required to enter a first password, then selected characters from a second password. KeyChainDD is built to be THE most secure password manager available for Mac OS X. Here are some of the reasons why you should use KeyChainDD:

KeyChainDD uses the Mac OS X keychain management infrastructure. KeyChainDD is built on top of the OS X keychain system, so unlike programs that create their own storage for sensitive data, compromising KeyChainDD won't give access to all you sensitive data. In fact, KeyChainDD never sees your master keychain password; you enter that directly into an OS X dialog when required.

KeyChainDD is open source. You, or anyone else, can examine all of KeyChainDD's code and verify that it is really is secure, has no hidden trap doors, etc.

KeyChainDD is built to support modern sites with double layer "memorable information" security systems. KeyChainDD easily allows you enter randomly selected characters from a second password, as required by many financial services web sites.

KeyChainDD allows you to easily use "part memorized" passwords. With part memorized passwords, you use KeyChainDD to store most of a long password, but manually add a few characters that you remember. This gives the security advantages of long passwords, but also defeats most malware.

KeyChainDD never "calls home". Unlike programs that contact a central server for license verification, to check for updates, or whatever else KeyChainDD never accesses the net. Ever.

For more information on how to use KeyChainDd securely, go to the KeyChainDD website.
